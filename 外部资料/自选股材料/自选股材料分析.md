## 3. 代码清洗规则（实施时务必遵循）

| 问题 | 规则 | 示例 |
|------|------|------|
| A股6位代码 | 保持原样，加交易所后缀 | `603019` → `603019.SH` |
| 港股5位代码 | 左侧补0到5位，加 `.HK` | `0700` → `00700.HK` |
| 美股字母代码 | 保持原样 | `AAPL`、`NVDA` |
| 东方财富板块BK | 不入自选股（数据源不直接支持），作为参考标签 | `BK0731` → 标记"农化制品板块" |
| ETF代码 | 保持原样，加后缀 `.{SH/SZ}` | `159262` → `159262.SZ` |
| 指数(000001)(000300) | 与A股代码冲突，需区分数据源 | `000001`→上证指数(INDEX) vs 平安银行(STOCK) |
| LOF（深市基金） | 16xxxx 加 `.SZ` | `161226` → `161226.SZ` |
| OCR错误校正 | 已有人工校正过的名称保持；`AI应用`(980112)等概念板块不做导入 | 见原始文件标注 |

## 4. MVP 阶段建议处理

1. **第一优先导入**：A股个股 ~105条（代码规整、数据源直接支持）
2. **第二优先导入**：场内ETF ~20条（A股交易所直接交易）
3. **第三优先导入**：港股 ~25条、美股 ~20条（需要多数据源或另配）
4. **暂时不导入**：板块指数(BKxxx)、东方财富概念指数、国际期货/黄金/白银、LOF/REIT（数据源可能不支持，且非股票标的）

预估 MVP 阶段自选股池：**~150条**（A股个股 + 场内ETF）。

剩余数据留待 v0.2 多数据源接入后扩展。
