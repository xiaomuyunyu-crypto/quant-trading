> 产品经理：窗口2 | 更新日期：2026-05-07 | 基于设计构想整理

## 1. 产品概述

### 1.1 项目背景

用户有基金实盘交易经验，但股票账户未开通。用户是趋势交易者，需要系统从个股/板块ETF入手（暂不碰基金净值估算），建立可用的量化辅助工具。

### 1.2 产品定位

以个股和场内ETF为标的的量化信号系统。核心能力：
- 外部数据源接入 → 读出K线数据
- 四个核心指标（量价理论/均线理论/MACD/RSI）在日/周/月线多周期分析
- 发出加减仓信号，投资者手动确认，系统记录跟踪
- 历史数据回测，持续修正指标权重
- 回测数据默认覆盖最近 1000 天，并支持从数据源可获取的上市首日开始回测

### 1.3 目标用户

单人使用 —— 项目发起者。单用户架构，无需登录/权限系统。

### 1.4 核心交易指标（4个）

| 指标 | 简介 | 用途 |
|------|------|------|
| 量价理论 | 成交量与价格配合关系 | 判断趋势强弱 |
| 均线理论 | 多根均价线（MA5/MA10/MA20/MA60） | 趋势方向、金叉死叉 |
| MACD | DIF/DEA/柱状图 | 日线：金叉/绿柱连续缩短3次/绿柱累计缩短4次→买入，死叉/红柱连续缩短2次/红柱累计缩短3次→卖出；周线/月线金叉死叉 |
| RSI | 相对强弱指标（0-100） | 反转观察：连续3日<20或连续2日<16买入；单日>92或连续2日>85卖出 |

### 1.5 多周期分析要求

日线、周线、月线各分配权重（可配置），综合得分决定信号强弱。

当前回测页展示 5 个 MACD 策略：

| 策略 key | 名称 | 规则 |
|----------|------|------|
| `triple_macd_ma250` | 原策略：月线+MA250+周线+日线 | 月线 MACD 过滤，MA250 长期开关，周线窗口；日线买=金叉或绿柱连续缩短3次或绿柱累计缩短4次，卖=死叉或红柱连续缩短2次或红柱累计缩短3次 |
| `triple_macd_no_monthly` | 松绑1：去掉月线MACD控制 | 去掉月线过滤，保留 MA250、周线窗口；日线按同一套绿/红柱缩短规则执行 |
| `triple_macd_no_monthly_no_ma250` | 松绑2：再去掉MA250控制 | 去掉月线和 MA250，保留周线窗口；日线按同一套绿/红柱缩短规则执行 |
| `triple_macd_daily_only` | 松绑3：仅日线MACD执行 | 去掉月线、MA250、周线控制；日线买=金叉或绿柱连续缩短3次或绿柱累计缩短4次，卖=死叉或红柱连续缩短2次或红柱累计缩短3次 |
| `weekly_macd_cross` | 周线MACD金叉/死叉 | 第5个策略保留简单规则：周线 MACD 金叉买入，周线 MACD 死叉卖出 |

### 1.6 系统持续性要求（硬约束）

- 信号必须有连续性：今天发出的信号与昨天有关联，不能反复无常（第一天买、第二天清仓）。
- 系统需具备**记忆机制**：记录历史操作和收益率，迭代优化策略，保留表现好的策略，剔除/优化表现差的。
- 用户是做趋势的，不是短线赌方向。

### 1.7 技术约束（已由总设计师确定）

| 项 | 选型 |
|----|------|
| 后端框架 | Python 3.12+ / FastAPI |
| 前端框架 | React 18+ / Vite |
| 数据库 | PostgreSQL + SQLAlchemy 2.0 |
| 数据源 | AKShare / Tushare（优先免费接口） |
| 任务调度 | APScheduler |

## 2. 功能模块

### 2.1 自选股管理（Watchlist）

用户创建和维护自选股票池。已有219只自选股材料（来源：东方财富"多股同列"截图 OCR 整理，含A股/港股/美股/ETF/指数）。

#### 2.1.1 功能列表

| 编号 | 功能 | 描述 |
|------|------|------|
| W-01 | 添加自选股 | 输入股票代码或名称搜索添加 |
| W-02 | 删除自选股 | 从自选池移除 |
| W-03 | 编辑备注 | 为每只自选股添加备注文字 |
| W-04 | 分组管理 | 创建/删除分组（如"蓝筹""短线""观察""港股通""美股"），股票可挂多分组 |
| W-05 | 列表展示 | 分页展示

#### 2.1.2 股票搜索输入体验

回测、行情查看、实盘模拟手动查看和后续自选股添加都应复用同一种股票搜索输入框：
- 用户可以输入 6 位股票代码，也可以输入中文股票名称。
- 输入中文关键字时，下方弹出候选列表，展示"市场 / 代码 / 名称"三列。
- 候选项点击后写入真实股票代码，后端回测仍以代码作为唯一标识。
- 搜索不依赖拼音首字母；第一版支持代码匹配、名称包含匹配、中文字符顺序模糊匹配，例如"万化"可以匹配"万华化学"。

### 2.2 实盘模拟（Paper Trading）

实盘模拟用于跟踪策略建议，但不自动下单。页面中区分两类股票池：
- 自选股：用户维护的股票池，可按需显式触发分析。
- 观察股：实盘模拟账户的当前持仓股，作为默认分析范围。

#### 2.2.1 信号生成范围

| 场景 | 范围 | 说明 |
|------|------|------|
| 默认分析 | 观察股（当前账户持仓） | 避免一次性分析全部自选股或全市场 |
| 自选股分析 | 自选股前若干只 | 需要用户显式点击触发，接口带上限 |
| 手动查看 | 搜索后选择的额外股票 | 支持股票代码和中文名称搜索候选 |

#### 2.2.2 信号展示

买入建议和卖出建议必须同时展示股票代码与股票名称。持仓面板也应展示名称，历史旧持仓名称为空时由股票基础表补全。

### 2.3 策略回测数据范围

回测页面默认使用最近 1000 天日线数据，并默认直连 AKShare 获取 K 线，避免线上 SQLite 缓存缺口影响回测。用户也可以选择“从上市以来回测”，后端以 `19900101` 作为历史抓取起点，请求 AKShare 后按实际可获取的首个交易日计算。

本地数据库已有近期数据时，数据管道不能只追加最新日期；如果请求区间早于本地最早 K 线，需要自动补齐历史缺口。命令行支持：

```powershell
python -m data.pipeline sync-klines --days 3000 --codes 600309
python -m data.pipeline sync-history --codes 600309
```
