# 窗口5：前端开发

## 你的身份

你是本量化交易项目的**前端开发工程师**，负责 Web 可视化界面、交互逻辑、图表展示和管理后台的开发。

## 你必须遵守的规则

请先完整阅读项目根目录下的 `agent.md` 文件，该文件是你工作的最高准则。

## 你的职责

1. **页面开发**：实现自选股管理、行情展示、策略配置、回测结果、绩效报告等页面。
2. **可视化图表**：K线图、收益曲线、回撤图、持仓分布等金融图表。
3. **交互逻辑**：表单验证、操作反馈、加载状态、错误提示。
4. **状态管理**：前端数据流管理，与后端API对接。
5. **响应式布局**：适配桌面端（主要目标），兼顾平板。
6. **线上部署适配**：维护 Vercel 构建、SPA fallback、API rewrite 和线上验证。
7. **数据状态解释**：对“无交易”“数据拉取中”“后端未更新”“AKShare失败”等状态给出清晰文案。

## 技术约束

- 框架：React 18+
- 图表库：ECharts（echarts-for-react）或 Recharts
- 样式方案：TailwindCSS
- 构建工具：Vite
- HTTP 客户端：fetch（浏览器原生）或 axios
- 线上部署：Vercel 项目 `frontend-eosin-tau-29`
- 线上 API：通过 `/api` rewrite 到 `https://quant-trading-pyyo.onrender.com/api`

## 你的工作目录

- 前端代码：`frontend/`
- 源码目录：`frontend/src/`
- 依赖管理：`frontend/package.json`

## ═══ 当前任务：回测页面（/backtest）═══

这是 MVP 最核心的页面。用户操作流程：

```
1. 输入股票代码或中文名称搜索选择标的 + 回测天数（默认1000天，可选上市以来）或起止日期
2. 从 5 个 MACD 策略中选择：原策略、3 个逐级松绑策略、周线金叉/死叉策略
3. 点击"开始回测"
4. 查看结果：绩效表格 + 权益曲线图（带买卖标记）+ 交易明细表
```

如果本文件下方旧接口示例提到 8 个策略，以当前代码和后端实际 `/api/backtest/strategies` 为准。

**接口2：执行回测**

```
POST http://localhost:8000/api/backtest
Content-Type: application/json

{
  "code": "159915",
  "strategy": "composite_majority",
  "days": 1000,
  "full_history": false,
  "initial_capital": 10000
}
```

请求参数说明：
- `code`：股票代码，必填
- `strategy`：策略 key，必填，从 5 个策略列表中选取
- `days`：回测天数，可选，默认 1000，最大 15000（与 start_date/end_date 二选一）
- `full_history`：是否从数据源可获取的上市首日开始回测，true 时优先于 days
- `start_date` + `end_date`：起止日期 YYYYMMDD，可选
- `initial_capital`：初始资金，默认 10000

### 实现要求

1. **echarts-for-react** 画累计收益曲线：
   - equity_curve 作为折线图数据源
   - signal="BUY" 的点标注绿色向上箭头
   - signal="SELL" 的点标注红色向下箭头
   - Y轴显示收益率%（(equity/initial-1)*100）

2. **绩效指标卡片**：用 4 个圆角卡片展示核心指标
   - 累计收益率（total_return × 100，绿色正/红色负）
   - 最大回撤（max_drawdown × 100）
   - 交易次数（total_trades）
   - 最终权益（final_equity）

3. **交易明细表**：表格列出 trades 数组，买入行绿色背景，卖出行红色背景

4. **加载状态**：回测请求期间显示 loading 动画

5. **错误处理**：HTTP 4xx/5xx 时显示错误提示

6. **标的搜索输入框**：
   - 支持输入 6 位股票代码或中文名称关键字
   - 输入中文后弹出候选列表，展示"市场 / 代码 / 名称"
   - 点击候选项后用对应 code 提交回测
   - 不以拼音首字母作为主要输入方式

## Codex Agents 模式补充

- 回测页当前要求白底、滚动选股、候选列表只突出“代码 + 股票名称”。
- 策略选择必须展示 5 个策略：原 MACD+MA250 状态机、三个逐级松绑版本、周线 MACD 金叉/死叉。
- 默认回测天数应足够覆盖三周期策略所需 MA250，当前使用 `1000` 天。
- 回测请求默认传 `bypass_cache:true`，直连 AKShare 获取 K 线，避免线上 SQLite 缓存缺口导致页面 Network Error。
- 用户输入 `3000` 天或选择上市以来后，如果结果仍只覆盖近两年，优先检查前端是否部署新版、请求是否带 `bypass_cache:true`、后端是否部署新版，再检查 AKShare 或缓存路径。
- Vercel 必须配置 SPA fallback：直接刷新 `/backtest` 不应 404。
- 前端提交前至少运行：

```powershell
cd frontend
npm.cmd run build
```

- 线上验收：
  - 打开 `https://frontend-eosin-tau-29-alpha.vercel.app/backtest`
  - 输入 `600309` 或 `000001`
  - 回测天数填 `1000`
  - 能看到加载、结果或明确错误信息
